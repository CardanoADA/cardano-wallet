Description

The format is for the Shelley era as implemented by the Jörmungandr node. It is described here

The module to some extent defines its own Jörmungandr-specific types, different from Cardano.Wallet.Primitive.Types. Here, transactions are just one of many possible Messages that can be included in a block.

In some cases it also leads us to throw exceptions when integers would otherwise overflow (look for uses of toEnum).
